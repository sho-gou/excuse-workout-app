# README

## 1. サービス概要
- **このサービスはどんなサービスか**
  仕事や付き合いで毎日忙しく働く30〜50代のサラリーマンをターゲットにした、「サボった言い訳をあえて記録・可視化して、挫折を防ぐ」超低ハードルな筋トレ継続支援サービスです。
- **誰の、どんな課題を、どう解決するか**
  「健康のために筋トレを始めたいけれど、残業や飲み会を言い訳にいつも三日坊主で終わってしまう」というビジネスパーソンの課題を解決します。筋トレの成果だけでなく「サボった言い訳」も1タップでスタンプ感覚で記録できる仕組みにすることで、サボり癖を客観的に自覚してもらい、完全に辞めてしまうのを防ぎます。

## 2. このアイデアはどこから生まれたか
自身が過去に何度も筋トレに挑戦するも、すべて三日坊主で終わってしまい、どうしても継続できない強い劣等感や悔しさから生まれました。

### 2-1. きっかけとなった体験・感情
健康診断の結果を見て、自分の体型の変化や将来の健康リスクにショックを受け、「今度こそ本気で筋トレを頑張るぞ」と決意してスケジュールを立てました。しかし、いざ開始すると「今日は仕事が遅かったから」「急な飲み会が入ったから」と、その日の忙しさを言い訳にして1日サボってしまい、そこから雪だるま式にやらなくなり、結局いつも通り挫折してしまいました。その度に「また続けられなかった」「このままでは自分に甘い、だらしないお父さんになってしまう」という強い自己嫌悪と焦りを抱いていました。

### 2-2. なぜそれが「気になった」のか
30代を過ぎ、年齢的にもここで体型や健康管理を諦めたら、本当に年齢を言い訳にする「だらしない大人（父親）」になってしまうという強い危機感があったからです。また、自分の性格として「やると決めた時の理想は高い（スケジュールを完璧に組む）」のに、「いざ実行する段階になると目先の言い訳に負けてしまう」という、理想と現実のギャップに強い違和感と課題感を持っていたためです。

## 3. 課題の整理（表面的になっていないか）

### 3-1. 表に見えている困りごと
- 過去の服が入らなくなる、見た目が悪くなる
- 体力の低下を感じる
- 筋トレのスケジュールを立てても長続きしない

### 3-2. 本当に解決したい課題は何か
- なぜ筋トレが続かないのか？ ➡️ 仕事の疲れや飲み会など、その日の「もっともらしい言い訳」が目の前にあると、筋トレの優先度を簡単に下げてしまうから。
- なぜ簡単に下げてしまうのか？ ➡️ 1日サボることのペナルティがなく、自分がどれだけ言い訳を積み重ねているかを客観的に自覚（視覚化）していないから。
- なぜ自覚していないと困るのか？ ➡️ 「明日からやればいいや」と毎日自分を甘やかし続け、結果として理想の自分から遠ざかり、自己嫌悪を繰り返す悪循環から抜け出せないから。

**➡️ 本当に解決したい課題：**
「目先の忙しさや付き合いを言い訳に自分を甘やかし、筋トレをサボってしまう心理的ハードルを越えられず、継続的な健康管理や理想の体型維持を挫折してしまうこと」

## 4. 想定ユーザーについて

### 4-1. 想定しているユーザー
- **年齢設定：** 30代〜50代の男性（特に既婚・子持ちのビジネスパーソン）
  - **理由：** 代謝が落ちて体型の崩れや健康診断の結果がリアルに気になり出す年代であり、同時に「子供や家族にとってカッコいい父親でありたい」という動機が生まれやすい世代だからです。
- **生活・状況設定：** 残業や突発的な飲み会、休日出勤など、自分の意思だけでは時間をコントロールしにくいサラリーマン。
  - **理由：** 時間が潤沢にある学生などとは違い、「忙しさ」という強力かつ正当な「サボる言い訳」が日常的に発生するため、本アプリの「言い訳を逆手に取って継続させる仕組み」が最も必要とされるからです。
- **環境設定：** 基本は自宅での自重トレーニング（スクワット、腕立て伏せなど）、または通勤中などの隙間時間。
  - **理由：** ジムに通う時間的・心理的ハードルが高い環境だからこそ、自宅で「1分でもやればOK」と思える超低ハードルな受け皿をアプリ側で用意し、実行のハードルを解消します。

### 4-2. 自分とユーザーの距離
- **自分自身**
  - **理由：** 自分自身が「仕事の忙しさ」を理由にして、何十回も筋トレを挫折してきた当事者だからです。ユーザーがどの瞬間に誘惑に負け、どんな言い訳を自分自身に対して行うのかが痛いほどリアルに理解できるため、この距離感を選びました。

### 4-3. 実際に使われる可能性について(需要・存在確認)
- **まず最初に使ってくれそうな人は誰か？**
  自分と同じように、健康診断の結果に焦りを感じて筋トレを始めたものの、残業や飲み会を理由に1週間以内に挫折してしまった同世代（30〜50代）の同僚サラリーマン。
- **その人は、どんな場面でこのアプリを思い出すか？**
  夜、仕事が終わってクタクタで帰宅した時、または飲み会から帰ってきたベッドの中で、「あ、今日も筋トレやれなかったな…」と罪悪感を覚えた瞬間。
- **今の生活の中で、代わりに使っているものは何か？**
  - スマホのカレンダーアプリやリマインダー
  - ノートや手帳への手書き記録
- **それを置き換えてまで使う理由はあるか？**
  既存のツールはすべて「頑張ってやったこと」を記録するものであり、サボった時の受け皿がありません。本アプリは、サボった時に「仕事が忙しかった」「飲み会だった」という言い訳を1タップで記録でき、蓄積された言い訳をグラフやカレンダーで突きつけられる唯一のサービスだからです。「サボった罪悪感を可視化し、明日への強制力に変える」という体験は、他のツールでは替えが利きません。

### 4-4. ユーザーがサービスを導入して利用するまでのイメージ(導入・継続の流れ)
- **どんな場面・困りごとがきっかけで、このサービスを知る・思い出すか**
  健康診断の再検査通知が届いた時や、久しぶりに着たスーツのズボンがきつくなっていた時。「今度こそ本気で続けたい、でもまた三日坊主になるのが怖い」とネットで「筋トレ 継続 コツ」「三日坊主 対策 アプリ」などを検索したことがきっかけで知ります。
- **最初の一歩（登録・導入）で、ユーザーは何をする必要があるか**
  1. Google認証などのSNSログインで10秒でアカウント作成。
  2. 最初に見る画面で、自分の目標（例：マイナス5kg、カッコいい父親になる）を設定。
  3. 「これだけは毎日やる」という超低ハードルなメニュー（例：スクワット10回）を1つだけ選択する。（※導入のハードルを下げるため、細かい体重入力や複雑なスケジュール設定はあえてさせません）
- **利用を続ける理由は何か？**
  - **心理的ハードルの低さ：** 疲れて帰ってきても、「やる」か「言い訳を選んでサボる」のどちらか1タップで今日の記録が終わるため、アプリを開くこと自体の負担がありません。
  - **言い訳カレンダーの心理的抑止力：** カレンダーが「残業」「飲み会」といった言い訳のスタンプで埋まっていくのを見ると、「自分はこんなに言い訳をしていたのか」と恥ずかしくなり、「今日は5回だけでもスクワットをして青（達成）にしよう」という行動の変化が生まれます。
  - **同世代とのゆるい連帯感（※本リリース以降の想定）：** 自分と同じように「仕事が忙しくてサボりそうになっているお父さんサラリーマン」が、言い訳しながらもなんとか踏ん張っているタイムラインを見ることで、「自分だけじゃない」とモチベーションが持続します。

## 5. 既存サービス・競合調査

### 5-1. 似たサービスの調査
- **サービス名：** 腕立て伏せ継続特化型の記録ツール（note等で紹介されているシンプルな継続支援アプリ）
- **URL：** https://note.com/pushup_habit/n/n6e8733c6e903
- **どんなことができるか：**
  「今日の腕立て伏せをやったかどうか」だけに特化し、余計なデータ入力を省いてシンプルに日々の実行を記録・カレンダー等で可視化できる。

### 5-2. それでも自分が作りたい理由
既存の多くの筋トレアプリは「ベンチプレス〇kgを〇回」といった本格的な数値入力が必要で、仕事終わりの疲れた頭にはその入力作業自体がハードルになり、結果として三日坊主に繋がっていました。
一方で、上記のような「やったかどうかだけを記録するシンプルアプリ」は素晴らしいですが、「本当に仕事や飲み会で疲れてサボってしまった日」にアプリを開く動機がなくなってしまうという点に物足りなさを感じました。多くのアプリは「サボった日は何も記録しない（空白になる）」ため、一度サボるとそのままアプリを開かなくなり、フェードアウトしてしまいます。
私は「やった日」だけでなく、「サボって言い訳をした日」も同じように1タップで肯定的に（あるいは自戒を込めて）記録できる場所が欲しいと考えました。サボった時の受け皿がないという既存サービスの“ズレ”を解消したいです。

### 5-3. 差別化を一文で決める
このアプリは、【仕事や付き合いで忙しいサラリーマン】の【疲れて筋トレをサボりそうになった瞬間（サボってしまった瞬間）】を一番助けるアプリです。

- **私のアプリの差別化ポイント：**
  「サボった言い訳」をスタンプ感覚で記録でき、蓄積された言い訳を可視化することで、自分のサボり癖を客観視して次の日の『超低ハードルな1歩』に繋げる「生存・継続特化型」の仕組み。

## 6. このサービスで提供したい価値

### 6-1. ユーザーの変化
このサービスを使うことで、ユーザーは以下のように変化します。
- **行動の変化：** 
  仕事や飲み会で疲れて帰ってきた日でも、アプリを開いて「1タップでサボった言い訳を記録する」という行動が習慣化します。さらに、言い訳が溜まる（体型が崩れる）のを防ぐために「スクワット5回だけでもやってから寝よう」と、小さな行動を起こせるようになります。
- **気持ちの変化：** 
  サボってしまった日も「言い訳の記録」という形でアプリに受け入れられるため、挫折に伴う過度な自己嫌悪（「どうせ自分は三日坊主だ」という諦め）が和らぎ、「明日またリスタートしよう」と前向きに捉えられるようになります。
- **考え方の変化：** 
  「完璧なスケジュールをこなすこと」だけが継続ではなく、「サボりながらも、自分の言い訳を自覚しつつ、完全に辞めずに生存し続けること」こそが真の継続であると気づくことができます。

### 6-2. 価値を一文で表す
「このサービスは【いつも忙しさを言い訳に筋トレを挫折してしまう人】が、【自分の言い訳癖を可視化し、体型の変化を楽しみながら、三日坊主を乗り越えて健康管理を続けられるようになる】サービス」

## 7. このアプリで実現すること

プログラミング初学者として、まずは最速で「サービスの核（記録と言い訳による体型変化）」を検証できる最小限の機能（MVP）から実装し、その後ユーザーの反応を見て発展させます。

### MVPで作る機能
1. **ユーザー管理機能**
   - ユーザー登録、ログイン、ログアウト（Sorceryを用いたメールアドレスまたはGoogle認証ログイン）
   - プロフィール編集（目標の設定、超低ハードルなメニューの選択）
2. **日々の生存記録（実行 / 言い訳）機能**
   - 1日1回、カレンダー上の「今日の枠」または専用ボタンから1タップで記録。
   - 「実行」ボタン：やったメニューを選択して登録（DB上には `status: "done"` として保存）。
   - 「サボる」ボタン：あらかじめ用意された言い訳スタンプ「残業」「飲み会」「体調不良」「なんとなく」等を選択して登録（DB上には `status: "excuse"` として保存）。
3. **言い訳カレンダー機能（ダッシュボード）**
   - 記録結果をカレンダー上で一覧表示（Gem `simple_calendar` を想定）。
   - 「実行した日」は青、「サボった日」は赤（または選んだ言い訳のスタンプアイコン）で視覚的に一目でわかるように表示し、同月内での行動パターンを可視化。
4. **サボり連動型アバター変化機能（★コアギミック）**
   - マイページに表示されるお父さんキャラクター（アバター）のグラフィックが、直近のサボり回数（`status: "excuse"` の数）に応じて条件分岐でリアルタイムに変化。
   - （例：連続サボりなし ＝ 普通、3回サボり ＝ お腹ふっくら、5回以上サボり ＝ ビール腹、逆に実行が続けば細マッチョに変化）

### 本リリースで作る機能
1. **言い訳分析（グラフ化）機能**
   - 今月自分がどの言い訳（残業、飲み会など）を何回使ったかを円グラフ等で表示し、自分のサボり傾向を客観的に自覚させるダッシュボード。
2. **タイムライン機能（SNS連携・ゆるい連帯感）**
   - ユーザー同士が「今日も残業でサボりました…」「出張だけどスクワットだけやりました！」とお互いの生存報告（言い訳含む）を共有し、匿名で「お疲れ！」スタンプを押し合えるゆるいコミュニティ機能。
3. **通知（リマインド）機能**
   - 夜の指定した時間に、LINE Bot等を通じて「今日の生存報告（実行または言い訳）はもう済んだ？」と通知を飛ばし、アプリの立ち上げを促す機能。

## 8. このアプリの懸念点とその対策

### 懸念点①
- **何が問題になりそうか：** 
  「サボった言い訳を記録する」という仕様上、ユーザーがサボることに慣れてしまい、最終的に筋トレ自体を全くやらなくなってしまう（目的のすり替わり）。
- **なぜそう思うか：** 
  アプリを開いて言い訳をタップするだけで「記録した満足感」が得られてしまうため、本来の目的である「筋トレによる健康管理」が進まない可能性があるからです。
- **そのために考えている対策：** 
  サボり（言い訳の記録）が積み重なると、マイページのアバターが目に見えて「ビール腹のだらしない体型」へリアルタイムに変化していく仕様にします。これにより、「このままではダメなお父さんになってしまう」という視覚的な危機感をユーモラスに刺激し、ユーザーが「ハッとする瞬間」を作ることで、再び「実行（筋トレ）」ボタンを押して元の体型（あるいは細マッチョ）に引き戻したくなる行動を促します。

## 9. 今後の展開・発展の方向性
- **ユーザーはなぜ使い続ける？：** 
  カレンダーのマス目を埋める（または赤を増やさない）というコレクション欲求と、自分の生活習慣（言い訳データ）が蓄積されていくライフログとしての楽しさがあるためです。また、アバターを「良い状態」で維持したいという育成ゲームのような愛着が湧くためです。
- **データが溜まると何ができる？：** 
  「自分が一番サボりやすい曜日（例：木曜の残業）」や「サボる引き金になるイベント（例：金曜の飲み会）」が統計データとして見えてくるため、生活習慣そのものの見直しや対策（例：水曜の朝に前倒しで筋トレをする等）に活かすことができます。
- **機能以外の工夫：** 
  初学者の個人開発として運用コストを抑えるため、最初は複雑なサーバー維持費がかからない構成にし、SNS（Xなど）で「#サラリーマンの言い訳筋トレ」といったハッシュタグで開発の裏側を発信することで、初期のユーザー（同じ悩みを持つお父さんサラリーマン）を巻き込んでコミュニティを作ります。

## 10. 技術スタック（手段としての技術）

### 10-1. 使用予定の技術
- **フレームワーク：** Ruby on Rails 7.x
- **DB：** MySQL
- **デプロイ先：** AWS (EC2 / Nginx)
- **主要な使用予定ライブラリ（Gem）：**
  - `sorcery`（認証機能）
  - `simple_calendar`（カレンダー表示）
  - `Tailwind CSS`（フロントエンドのデザインUI調整）

### 10-2. 技術選定の理由
- **なぜこの技術を使うのか：** 
  基礎を深く学習した「Ruby on Rails」および「MySQL」をベースにすることで、実装のバグを最小限に抑え、サービスのコアロジック（実行とサボりのCRUD処理、アバターの画像切り替え）の実装に集中するためです。
- **今回チャレンジしたい点：** 
  `simple_calendar` の標準機能を利用し、カレンダーのマス目内に保存された `status`（done / excuse）に応じた条件分岐を組み込み、直感的に自分の継続状況がわかるUI/UXを構築すること。
- **不安な点：** 
  カレンダー画面の中で、日付ごとに「実行データ」と「言い訳データ」を綺麗に出し分けるコントローラーとビューの条件分岐ロジック。これに関しては、事前に小さなモック（試作品）を作って検証します。

### 10-3. キャッチアップ不足の懸念
最近モダンなフロントエンドとして注目されている「React」や「Next.js」について、最初から導入するとキャッチアップに多大な時間を取られ、MVPのリリース（完成）自体が遅れるリスクがあります。
そのため、**MVPおよび初期リリース段階ではRailsの標準機能（Hotwire/Turbo）とシンプルな条件分岐（if文）でアバター変化などを構築**し、アプリがしっかりとユーザーに受け入れられ、技術的な検証を済ませた後の発展形として、本リリース以降にフロントエンドのReact化を検討する方針とします。
